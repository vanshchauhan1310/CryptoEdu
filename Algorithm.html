Cryptographic Algorithms

AES (Advanced Encryption Standard)

A symmetric block cipher used for secure data transmission.

AES operates on 128-bit blocks and supports key sizes of 128, 192, and 256 bits. It's widely used for government and commercial applications.

RSA (Rivest-Shamir-Adleman)

An asymmetric cryptographic algorithm used for secure data transmission and digital signatures.

RSA uses two keys: a public key for encryption and a private key for decryption. Its security is based on the difficulty of factoring large prime numbers.

Blowfish

A symmetric block cipher designed as a fast, free alternative to existing encryption algorithms.

Blowfish uses a variable-length key, from 32 bits to 448 bits, making it ideal for both domestic and exportable use.

SHA-256 (Secure Hash Algorithm 256-bit)

Part of the SHA-2 family, widely used for data integrity verification.

SHA-256 produces a 256-bit (32-byte) hash value, typically rendered as a 64-digit hexadecimal number. It's commonly used in cryptocurrencies like Bitcoin.

MD5 (Message Digest algorithm 5)

A widely used hash function that produces a 128-bit hash value.

While MD5 is still used for non-cryptographic purposes, it's considered cryptographically broken and unsuitable for security applications.

bcrypt

A password hashing function designed to be slow and resist rainbow table attacks.

bcrypt incorporates a salt and is adaptive, allowing the iteration count to be increased to make it slower as computers get faster.
